使用hexo+github免费搭建个人博客网站

学习笔记
  • Hexo
  • github


第一篇文章记录一下自己使用hexo+github搭建博客的过程

步骤

  • 安装 Git
  • 安装 Node.js
  • 安装 Hexo
  • 初始化 Hexo
  • 部署到 Git
  • 安装主题
  • 创建文章

安装 Git

  • 下载git安装包

Git-2.19.0-32-bit.exe
Git-2.19.0-64-bit.exe

  • 检测安装是否成功

打开cmd命令行(win+r 输入cmd回车)执行 git –version

出现图片里的信息就说明安装成功

安装 Node.js

  • 安装 Node.js安装包

node-v8.12.0-x86.msi(32位)
node-v8.12.0-x64.msi(64位)

  • 检测安装是否成功
npm会随着Node.js自动安装

打开cmd命令行(win+r 输入cmd回车)分别执行 node -v 和 npm -v


出现图片里的信息就说明安装成功

安装 Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

初始化 Hexo

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

新建完成后，指定文件夹的目录如下：

  • _config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

  • source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

  • themes

主题 文件夹。Hexo 会根据主题来生成静态页面。我们下载的主题会放在这个文件夹

部署到 Git

  • 创建GitHub仓库

在github上新建一个名为 YourgithubName.github.io 的远程仓库, 例如填写wapchief.github.io以后, 在地址栏直接输入wapchief.github.io就可以访问网站

  • 部署

修改我们的的站点配置文件_config.yml中如下字段

其中：

  • type 部署的服务器类型，这里填写git就可以。
  • repo 是你的仓库地址, 填写刚刚创建的仓库克隆地址
  • branch 默认值为master

以上配置完成后保存 然后项目根目录执行下面的命令安装一个插件

安装完成后在终端中依次执行如下代码

  • hexo clean 清理缓存
  • hexo generate 简写 hexo g 进行渲染, 会生成静态文件
  • hexo server 简写 hexo s 启动服务器。默认情况下，访问网址为： http://localhost:4000
  • hexo deploy 简写为 hexo d 部署到git

就可以用 YourgithubName.github.io 来访问你的博客了
